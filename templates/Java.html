Introduction

Java is an object oriented programming language. All Java programs contain at least one class and it's name must match the name of the file. Every Java program must contain a method called main that contains the instructions for the program. Comments syntax is the same than JavaScript: // for singe line and /* ... */ for multiline.
Java does not interpret whitespace and statements are terminated with semicolons ';'.

Compiling java programs

We dont need to worry about compiling programs because the machine will do it for us but if you still want to compile something run javac fileName.java and a filename.class will be created (I think this is a bytecode file, or something like that). This file can then be run using java fileName.

Data types and variable declaration

Variable names start with a letter, $ or underscore. For declaring variables we need to specify what data type they are:

int age = 27;

Data types are:

  • int: integers.
  • double: decimals.
  • boolean: true / false (lowercase).
  • char: holds single characters surounded by single quotes. 'A'.
  • String: strings (note the capital S).

String methods

  • .length() returns the number of characters.
  • str1.concat(str2) joins the strings together, but you can also do str1 + str2.
  • str.indexOf(substring) returns the index of the substring or -1 if not found.
  • .charAt(index) returns the character at the spexified index.
  • .substring(start, end) returns the substring between start and end-1, omit second argument to go till the end.
  • .toUpperCase() returns the string in UPERCASE. Does not modify original string.
  • .toLowerCase() returns the string in lowercase. Does not modify oringinal string.

Arrays

Arrays do everything we expect them to do. They can store all data types and other objects; but this is Java, remember? and that means all elements in an array have to be of the same data type and it's size is fixed. To declare an array use:
type[] varName = {value1, value2, ... , valuen};
When working with arrays, import the Arrays package at the top of the file to get access to methods like Arrays.toString() which prints out arrays in human readable format: import java.util.Arrays;
To access an element in an array by index use the good old square bracket syntax: type elementOne = array[1];

To initialize an empty array, we have to set it's size:
type arrayName = new type[size];
Where size is an integer.

Arrays are objects and they have a property called length that stores magic spells (no, it's length).

Array Lists

Arrays hvave a fixed length, which means you can't add or remove elements from them. Luckily, there exists the ArrayList, which is exactly the same but without a fixed length.
To use them first we have to import java.utils.ArrayList them

ArrayList<String> myArrayList = new ArrayList<String>();

myArrayList.add(stuff);
>> Adds stuff to the end of the Array List.

myArrayList.size();
>> Returns the size of the Arra List.

myArrayList.get(index);
>> Returns the element at index.

myArrayList.set(index, newValue);
>> Rewrites the element at index to be newValue.

myArrayList.remove(element);
>> Removes the first instance of element.

myArrayList.indexOf(element);
>> Returns the index of element.


Note: for some Java reason, when creating ArrayLists made of chars, ints, or doubles, we have to write Char, Integer, or Double instead of String.

Inheritance

Use the keyword extends to create a subclass. Inside the subclass' constructor method, the built-in super() method acts as the parent's constructor.
Use the final keyword before the access modifier in a parent's class method to avoid any child classes overriding that method.
To override a method, use the @Override keyword in the line before the method to override. Also note it needs to have the same return type and same number and type of parameters.
Child class instances can be used where parent class objects are expected. This means different type of objects can go in an array as far as they belong to the same family.
